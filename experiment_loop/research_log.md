# research_log.md — provenance for web-sourced hypotheses

You may search the web for feature *mechanisms*. Every web-sourced idea is logged here so the provenance
chain holds and the ledger is replayable in spirit even though the open web is not. Provenance rule:
cite the **successful** fetch (URL + access date); never quote a statistic from memory.

Append a block per sourced idea:

```
## <date> — <one-line hypothesis>
Source:   <URL>            (accessed <date>; quote the exact claim used)
Claim:    <the mechanism/fact taken from the source>
Use:      MECHANISM        # built from the vehicle's own history -> free, referee-judged
       or EXTERNAL_NUMBER  # a fetched quantity -> data, not a feature: as-of/PIT + parity +
                           #   quarantine (negative-control arm) until proven entity/time-clean
Feature:  <the column(s) it became, and which arm/candidate>
Status:   LOGGED
```

(Empty until the agent runs. Web access is granted at launch — the loop's coding agent must be started
with WebSearch/WebFetch tools enabled; this file governs how it uses them.)
